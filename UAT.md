# บทที่ 5 การทดสอบระบบ

## 5.1 วัตถุประสงค์ของการทดสอบ

การทดสอบการยอมรับระบบจากผู้ใช้ หรือ User Acceptance Test (UAT) มีวัตถุประสงค์เพื่อประเมินว่าเว็บไซต์ `An Interactive Website for Learning Quantum Computing` สามารถทำงานได้ตรงตามความต้องการของผู้ใช้ในมุมมองการใช้งานจริง โดยเน้นการตรวจสอบความถูกต้องของการนำทางระหว่างหน้า การเรียนรู้แบบ interactive การจำลองวงจรควอนตัม การแสดงผลบน Bloch Sphere การเรียนรู้อัลกอริทึมควอนตัม และการทำแบบฝึกหัดท้ายบท

ผลจากการทดสอบในส่วนนี้จะใช้ยืนยันว่าระบบมีความพร้อมสำหรับการใช้งานในเชิงการเรียนรู้ สามารถช่วยให้ผู้ใช้งานเข้าถึงเนื้อหาได้อย่างต่อเนื่อง และรองรับกิจกรรมหลักที่กำหนดไว้ในขอบเขตของโครงงาน

## 5.2 ขอบเขตการทดสอบ

การทดสอบในบทนี้ครอบคลุมโมดูลหลักของระบบดังต่อไปนี้

- หน้าแรกของระบบ (Home)
- หน้าเรียนรู้เนื้อหา (Learn)
- หน้า Quantum Circuit Simulator
- หน้า Bloch Sphere
- หน้า Algorithms
- หน้า Practice
- หน้า About

ส่วนที่ไม่อยู่ในขอบเขตของการทดสอบครั้งนี้ ได้แก่ ระบบล็อกอิน ระบบฐานข้อมูลเชิงลึก การทดสอบประสิทธิภาพภายใต้ผู้ใช้จำนวนมาก และการทดสอบความเข้ากันได้ของทุกเบราว์เซอร์ย่อยทุกเวอร์ชัน

## 5.3 สภาพแวดล้อมที่ใช้ทดสอบ

สภาพแวดล้อมที่เหมาะสมสำหรับการทดสอบมีดังนี้

- ระบบปฏิบัติการ Windows 10/11 หรือ macOS
- เว็บเบราว์เซอร์ Google Chrome, Microsoft Edge หรือ Mozilla Firefox เวอร์ชันปัจจุบัน
- ความละเอียดหน้าจอระดับ Desktop และ Mobile Responsive
- ต้องสามารถโหลดทรัพยากรภายนอกได้ เช่น Tailwind CDN, Lucide และ Google Fonts

หมายเหตุ: จากการตรวจสอบในสภาพแวดล้อมปัจจุบันพบว่า dependency ที่ระดับ root ของโปรเจกต์ยังไม่พร้อมสำหรับการ build ผ่าน `vite` จึงจัดทำเอกสารส่วนนี้ในลักษณะกรณีทดสอบสำหรับนำไปทดสอบจริงและบันทึกผลต่อในภายหลัง

## 5.5 เกณฑ์การผ่านการทดสอบ

เกณฑ์การผ่านของ UAT ในโครงงานนี้กำหนดไว้ดังนี้

- ผู้ใช้สามารถเข้าถึงทุกหน้าหลักของระบบได้จากเมนูหรือปุ่มที่เกี่ยวข้อง
- ฟังก์ชัน interactive ของแต่ละหน้าแสดงผลได้สอดคล้องกับการกระทำของผู้ใช้
- ไม่เกิดข้อผิดพลาดที่ทำให้ผู้ใช้ไม่สามารถเรียนต่อหรือทดสอบต่อได้
- กรณีทดสอบระดับสำคัญสูงต้องผ่านครบก่อนการส่งมอบงาน

สถานะที่ใช้ในเอกสารนี้ประกอบด้วย `Pending`, `Pass` และ `Fail`

## 5.7 สรุปผลการทดสอบ

จากการออกแบบกรณีทดสอบ UAT สำหรับระบบ `An Interactive Website for Learning Quantum Computing` พบว่าระบบมีโมดูลหลักที่สามารถแบ่งกรณีทดสอบได้ทั้งหมด 30 กรณี ครอบคลุมทั้งการนำทาง การเรียนรู้แบบ interactive การจำลองวงจรควอนตัม การควบคุมสถานะบน Bloch Sphere การเรียนรู้อัลกอริทึม และการทำแบบฝึกหัด

ในสถานะปัจจุบัน เอกสารฉบับนี้ถูกจัดทำในลักษณะ “รายงานกรณีทดสอบเพื่อใช้ทดสอบจริง” โดยผลการทดสอบจริงยังคงเป็น `Pending` เนื่องจากยังต้องนำเอกสารไปใช้ประกอบการทดสอบบนระบบที่พร้อมใช้งานจริงอีกครั้ง เมื่อดำเนินการทดสอบครบถ้วนแล้ว สามารถอัปเดตคอลัมน์ `ผลการทดสอบจริง` และ `สถานะ` ให้สอดคล้องกับผลลัพธ์ที่เกิดขึ้นได้ทันที

ตารางที่ 5.10 สรุปจำนวนกรณีทดสอบ

| สถานะ | จำนวน |
| --- | ---: |
| Pending | 30 |
| Pass | 0 |
| Fail | 0 |

## 5.8 ข้อเสนอแนะในการใช้งานเอกสารฉบับนี้

- หากต้องการส่งในรูปแบบรายงานเล่ม สามารถนำเนื้อหาในไฟล์นี้ไปจัดรูปแบบต่อใน Word ได้ทันที
- หากอาจารย์ต้องการรูปแบบคล้ายตัวอย่างรายงานแบบ `TC01`, `TC02`, `TC03` แยกคนละหน้า สามารถแตกแต่ละกรณีทดสอบออกเป็นหน้ารายงานย่อยได้
- หากมีการทดสอบจริงแล้ว ควรเพิ่มภาพประกอบผลลัพธ์หน้าจอของแต่ละกรณีทดสอบเพื่อเพิ่มความสมบูรณ์ของรายงาน
